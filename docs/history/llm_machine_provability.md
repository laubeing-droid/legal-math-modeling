# LLM 机器可证性历史说明

## 文件定位

这是一份历史方法论说明，讨论“哪些证明更适合转成机器可校验工件”。

它不是当前 theorem 状态清单，也不是当前 release claim。

## 仍然成立的核心观点

以下判断今天仍然有效：

1. 计算机可直接复核的证明最适合交给自动化流水线
2. 需要长链创造性推理的证明最容易出现夸大和漏洞
3. Lean、SMT、穷举、证书校验的价值在于把“看起来对”变成“可复核”

## 当前需要修正的地方

这份早期说明默认了很多“未来会补完”的乐观表述。当前仓库已经把这些口径收紧：

- Banach 不再作为 formal-core 已完成目标描述
- 机器可证性分析不再自动等于 release claim
- 当前公开口径以 `docs/formal-release/` 为准

## 当前正确结论

对本仓最有价值的不是“LLM 能证明多少”，而是：

- 哪些内容已经形成可复现的 release boundary
- 哪些内容仍然只是研究轨道
- 哪些内容必须禁止对外夸大
