# 03 数学模型详析

日期：2026-06-28（原版 2026-06-11）

## 1. 总体模型

juris-calculus 的数学模型采用分层结构，每层对应一组 Lean 形式化定理：

```
Legal Data / Sources
        |
        v
Source Manifest + Trust Labels
        |
        v
Monotone Horn Closure         (HornDefinitions.lean, HornFixedPoint.lean)
        |
        v
Dung Grounded Extension       (DungDefinitions.lean, DungAAF.lean, DungFixedPoint.lean)
        |
        v
Banach Contraction Layer      (BanachContraction.lean, BanachFixedPoint.lean)
        |
        v
Agent Payload with Epistemic Status (UnifiedModel.lean)
```

## 2. Horn 层

### 2.1 形式化定理

Horn 层的形式化基于以下 Lean 文件：

- `HornDefinitions.lean`：Horn 子句和闭包算子的形式定义
- `HornFixedPoint.lean`：闭包算子的不动点存在性定理
- `HornOperationalRefinement.lean`：操作精化关系
- `FiniteMonotoneIteration.lean`：有限单调迭代的收敛性

### 2.2 数学性质

- 正向事实扩张的单调性
- 有限知识库上的闭包可验证性
- 单调算子在有限域上的不动点存在性

### 2.3 工程接口

```python
horn_result = horn_closure(facts, rules)
```

输出包含：fired rules、closure set、rule maturity、source ids。

## 3. Dung 抽象论证层

### 3.1 形式化定理

- `DungDefinitions.lean`：论证、攻击、可接受性、grounded extension 的形式定义
- `DungAAF.lean`：抽象论证框架的核心性质
- `DungFixedPoint.lean`：grounded extension 作为不动点的存在性证明

### 3.2 数学性质

- Grounded extension 的存在性和唯一性（通过特征函数不动点）
- 可接受性论证的性质
- 攻击关系的传递性质

### 3.3 工程接口

```python
aaf_result = grounded_extension(arguments, attacks)
```

输出包含：grounded extension、rejected arguments、active attacks。

## 4. Banach 层

### 4.1 形式化定理

- `BanachContraction.lean`：压缩映射的定义和基本性质
- `BanachFixedPoint.lean`：Banach 不动点定理
- `BanachComplete.lean`：完备度量空间的性质
- `BanachWeightedNorm.lean`：加权范数的定义和性质
- `BanachEffectiveNodes.lean`：有效节点的分析
- `BanachCertificate.lean`：Banach 证书的定义
- `ContractionCondition.lean`：压缩条件的判定

### 4.2 数学性质

- 完备度量空间上压缩映射的唯一不动点
- 加权 sup 范数的完备性（WeightedSupNorm.lean, SupZeroLemma.lean）
- 迭代序列的误差界

## 5. 跨结构层

### 5.1 形式化定理

- `FiniteGaloisAdjunction.lean`：有限 Galois 伴随的性质
- `FiniteRosetta.lean`：有限结构间的 Rosetta 映射

### 5.2 工程规则

跨结构映射采用 obstruction-first 路由：

1. 先查 obstruction registry
2. 若 COLLISION 或 ASYMMETRY，禁止自动映射
3. 若 MATCH，仍需保留来源标签

## 6. 时序与统一层

- `TemporalKripke.lean`：时序 Kripke 语义的模态逻辑性质
- `UnifiedModel.lean`：整合 Horn、Dung、Banach 的统一模型
- `JC_Formalization.lean`：主形式化入口
