# 04 算法实现分析：落地接口版

日期：2026-06-11

## 1. 目标

本文件把数学模型转换为下一步代码提升任务。所有算法都必须以证据标签为边界，不能把草稿、toy、proxy 数据提升为生产证明。

## 2. 必做算法改造

### 2.1 Evaluator 分层

旧模式：

```text
facts + rules + rebuttal + exception + confidence-zeroing
        -> one fixpoint loop
```

新模式：

```text
Stage 1: monotone_horn_closure(facts, horn_rules)
Stage 2: build_attack_graph(closure, rebuttals, exceptions)
Stage 3: dung_grounded_extension(graph)
Stage 4: emit trust-labeled result
```

验收：

1. Horn 层单调；
2. AAF 层确定；
3. 原 evaluator 非单调反例进入回归测试。

### 2.2 Graph similarity 降级

不得作为 metric 或 kernel。

新名称建议：

`ContextualOverlapScore`

必须修复：

1. 空特征自相似策略；
2. 接口文档禁止 metric 语义；
3. 所有调用点避免依赖三角不等式。

### 2.3 Pricing estimator

输入必须包含：

1. `data_quality`
2. `billing_type`
3. `source_id`
4. `paired_observation_group`

若 `data_quality == fee_schedule`：

```text
status = PASS_WITH_EXPECTED_DOWNGRADE
allowed_claim = schema/sensitivity/proxy analysis only
```

若需要真实 Banach/calibration：

```text
require real_timesheet or invoice
```

### 2.4 DP policy loader

必须拒绝：

1. 没有 source_id 的 epsilon；
2. 声称 epsilon legally determined 的数据；
3. 没有 approval/audit 字段的策略。

### 2.5 Trust label payload

统一 payload：

```json
{
  "result": "...",
  "confidence": 0.0,
  "epistemic_status": {
    "trust_label": "...",
    "rule_maturity": "...",
    "data_origin": "...",
    "verification_artifacts": []
  },
  "agent_instruction": {
    "action": "...",
    "forbidden": []
  }
}
```

## 3. 回归测试清单

| 测试 | 目的 |
|---|---|
| nonmonotone evaluator counterexample | 防止把原 evaluator 当单调 |
| AAF 100 case validation | 防止 AAF fixture 编码退化 |
| source manifest referential integrity | 防止孤儿 source_id |
| synthetic/toy promotion guard | 防止合成数据变证明 |
| pricing insufficiency gate | 防止 proxy 变 real proof |
| DP epsilon gate | 防止法源推出 epsilon |
