用户现在需要审计 JC 数学模型中数学证明 / 形式化的全部极限，以及可推进的方向。首先得梳理清楚现有证明的边界、限制，然后分 “现有极限” 和 “可推进的方向（含新的极限）” 来组织。首先，先拆解现有数学证明 / 形式化的核心极限，要覆盖定理状态、证据等级、数据限制、工具链、假设局限、形式化范围等维度。然后再分析可推进的方向，每个方向能突破的点，以及推进后仍存在的新极限。首先，现有极限的分类： 定理证明状态的固有极限（PROVED/REFUTED/PENDING 的边界） 形式化范围的极限（Lean4 的 sorry、仅覆盖纯 Horn closure） 证据等级与数据的极限（toy proof、proxy 数据、样本量） 数学假设的极限（单调性、度量性质、隐私机制） 工具链与工程实现的极限（缺失目录、未集成模块） 跨域 / 实证的极限（映射 obstruction、校准无负样本） 然后，可推进的方向，每个方向对应的推进点，以及推进后仍有的极限： 比如 Lean4 消除 sorry，但仍有高阶定理的形式化难度；扩展形式化到 temporal 模块，但仍有时间复杂度限制；MDL 的因果验证，但仍有样本量限制；等等。现在开始组织内容，确保覆盖所有维度，结构清晰，用 Markdown，分 “一、现有数学证明 / 形式化的全部极限” 和 “二、可推进的数学证明 / 形式化方向（含推进后仍存的极限）”。

三、核心结论：不可突破的终极极限
范围极限
：任何形式化 / 证明仅能覆盖 “纯 Horn closure + 有限集 + 单法域”，完整 evaluator / 无限集 / 跨法域的通用证明不存在（被反例 /obstruction 推翻）；
数据极限
：真实法律数据的隐私 / 合规限制，导致 “全域实证” 永远不可行，仅能基于样本 / 代理数据做经验验证；
复杂度极限
：高阶范畴论 / 时态逻辑的形式化受限于 Mathlib 能力与工程成本，无法在可接受时间内完成全量验证；
红线极限
：不得混淆 “toy proof / 相关 / 代理数据” 与 “全域定理 / 因果 / 真实实证”，这是不可突破的审计红线。
所有可推进方向均为 “在现有边界内深化”，而非 “突破核心极限”，核心价值是提升证明的严谨性、覆盖度与可审计性，而非推翻已验证的 REFUTED 结论或无证据的假设。

def violates_constraint (claim : Prop) (violation : ConstraintViolation) : Prop :=
  match violation with
  | MONOTONICITY_OVERREACH =>
    claim = (∀ (E : Evaluator), Monotone E) ∧
    ¬(E = PureHornEvaluator)
  | TOY_AS_UNIVERSAL =>
    ∃ (T : CoreTheorem), (theorem_metadata T).evidence = TOY_SYNTHETIC ∧
    claim = (∀ x, T x)
  | PROXY_AS_REAL =>
    ∃ (T : CoreTheorem), (theorem_metadata T).evidence = DATA_PROXY ∧
    claim = (∀ x ∈ RealWorld, T x)
  | PENDING_AS_PROVED =>
    ∃ (T : CoreTheorem), (theorem_metadata T).status = PENDING_TOOLCHAIN ∧

def advance (T : CoreTheorem) (e : EvidenceType) : CoreTheorem :=
  if (theorem_metadata T).status = PENDING_TOOLCHAIN ∧
     e ∈ {FINITE_EXHAUST, SYMBOLIC, SMT} then
    { theorem_metadata T with status := PROVED, evidence := e }
  else
    theorem_metadata T

-- ==============================================
-- 5.2 推进算子的边界保持性定理
-- ==============================================
theorem advance_preserves_boundary :
  ∀ (T : CoreTheorem) (e : EvidenceType),
    let T' := advance T e
    T'.domain_bound = (theorem_metadata T).domain_bound :=
by
  intro T e
  simp [advance]
  split <;> rfl

theorem advance_cannot_make_refuted_proved :
  ∀ (T : CoreTheorem) (e : EvidenceType),
    (theorem_metadata T).status = REFUTED →
    (advance T e).status = REFUTED :=

  intro T e h
